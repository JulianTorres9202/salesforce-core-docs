# Inclusive Language Updates in Business Manager

As we refine language in our products to more accurately reflect inclusivity, we replaced the word “master” with “main” or “base” in the Business Manager user interface and our documentation. To avoid any effect on customer implementations, we didn’t make any API changes.

Note The content in this document necessarily engages with non-inclusive language that can be emotionally and intellectually challenging to engage with.

Where: This change applies to all versions of Salesforce B2C Commerce and B2C Commerce services in Production, Sandbox, and Development instances.

When: Starting with the B2C Commerce 22.7 release.

Why: Equality is a core value at Salesforce. To ensure that language in Salesforce content and products is more inclusive, we continue to make updates and communicate them in the release notes.

How: We applied these changes:

-   master product / variation master –> base product
-   master and child items –> main items and secondary items
-   master catalog / master product catalog –> product catalog
-   master document –> main document

To learn more about terms that are changing, see [How we’re bringing inclusive language to our products](https://www.salesforce.com/news/stories/how-were-bringing-inclusive-language-to-our-products/).
